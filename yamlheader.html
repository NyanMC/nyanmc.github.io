this is the yaml header. this is a header that i type out for most yamls that i make, for the purposes of organization. it lists the multiworld the yaml was initially made for, the games it contains, and any other important information about the yaml, such as the presence of deathlink, energylink, item links, or plando. if there is something in the yaml which could harm the multiworld, it will contain an exclamation point before and after, sometimes multiple (e.g. ! Priority Vault !, !!! WARNING: Coinsanity 1 DLCQuest !!!, !!! WARNING: EXTREME numbers of Bee Traps !!!).

FAQ

But why?

i collect my yamls and store them in a "mini bucket", and this helps with organization. if i want to dig out a yaml in the future, it helps me recall what all the yaml even contains. if you want to view the contents of my "mini bucket", you can do so here, though i don't update it often.

Why does every slot end with {NUMBER}?

if i were to want to, say, generate every yaml in the bucket simultaneously, this would allow me to do just that. i sometimes end up needing to do this on a much smaller scale, for instance if i want to use universal tracker for several multiworlds at once (plus, doing all the yamls at once could make for a fun solo async sometime, you never know). if you are the host and this bothers you, you can strip it from the yaml and it will not affect me.
